# Wave 0 UI 占位任务演练记录

本文件是 Wave 0 多 Agent 流程演练的一部分，不代表真实前端功能已经实现。

## 可写边界声明

一个占位前端任务开始前，应在任务文件中明确任务 ID、所属 Wave、唯一可写路径和禁止触碰的路径。本演练任务的唯一可写路径是：

```text
docs/audits/dry-run-ui.md
```

任务执行者只能在自己的独立 worktree 中修改该文件；不得修改 `apps/web`，也不得修改任何其他文件、公共接口、共享 Schema、依赖或数据库迁移。若任务需要超出该边界，必须先停止并由主控 Agent 提交相应 RFC 或请求人工项目负责人批准，不能自行扩大范围。

## 失败报告方式

若占位任务无法完成，执行者应保留已观察到的证据，并在交接报告中说明：失败步骤、完整命令、失败输出或错误摘要、受影响的文件、是否产生了改动，以及需要谁作出决策。失败不应通过修改边界外文件、强制重置或覆盖未知改动来“修复”。如果是环境问题，也应区分“代码/文档失败”和“测试无法启动”，并给出可复现的后续动作。

本文件只记录流程约束；它不实现前端页面、组件、路由、接口调用或任何真实业务逻辑。

## 为什么必须由主控重新运行测试

子 Agent 的“已完成”声明不是验收结果。主控 Agent 需要在集成前重新运行任务要求的关键测试，以确认测试针对的是正确 worktree 和最终 diff，并排除本地环境、未提交改动、测试选择错误或子 Agent 误报造成的假通过。主控还应检查 diff 是否只落在声明的所有权边界内，再决定是否进入集成顺序。

对于本演练，文档不引入可运行的业务代码，因此不执行真实 UI 测试；主控仍应重新核对文件路径、内容和 Git diff，确认没有 `apps/web` 或其他越界文件变化。

## 演练范围声明

这是一次 Wave 0 流程演练。明确禁止修改 `apps/web` 以及除本文件外的任何文件；不得将本演练解释为真实前端任务或业务功能交付。
