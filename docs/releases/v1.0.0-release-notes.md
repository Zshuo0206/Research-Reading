# Research Reading Workbench V1.0.0 Release Notes（准备稿）

状态：`RELEASE_CANDIDATE_AWAITING_REAL_BYOK_ACCEPTANCE`

本文档是发布准备材料，不代表 V1.0.0 已发布，也不创建或推送 tag。

## V1.0 范围

- 本地单用户、loopback 部署。
- 文本型 PDF 导入、canonical page text 和页级回查。
- 快速问答。
- Guided Learning UNDERSTAND 阶段：学习方向、逐题问题、回答、点评、Reference Answer、Evidence 和阶段总结。
- Mock 生成和 OpenAI-compatible BYOK 生成路径。
- Session 刷新/重启恢复、RETRY、EDIT_ANSWER、PDF 恢复和 Evidence 页码定位。

## 安装和启动

使用仓库锁定的 Node 24/npm 依赖，在本地配置 SQLite/content root 和 loopback API/Web/Worker 进程；详细步骤见 `docs/operations/wave1-local-runbook.md`。

## BYOK 配置和安全边界

provider、HTTPS base URL、model 和 limits 由服务端接收；API key 只由 Worker 运行环境的 `WORKFLOW_BYOK_API_KEY` 提供。key 不进入浏览器、SQLite、Job、Session、response、日志、URL 或 Git。真实 BYOK 验收必须使用用户自己的安全凭据。

## 已知限制

- 当前 Release Gate 尚未完成真实外部 BYOK 和浏览器人工验收。
- Evidence 服务端执行 canonical source verification，不自动证明 claim 与 quote 的语义蕴含；需人工抽查。
- 仅支持文本型 PDF；不支持 OCR、扫描 PDF、复杂表格/公式、多篇论文。
- ANALYZE、TRANSFER、远程部署、多用户、知识图谱、写作辅助和团队协作不在 V1.0。

## 发布条件

自动化 Gate 已通过，但必须补齐真实 BYOK、浏览器人工验收、Evidence 人工抽查和产品负责人批准后，才可建议 `READY_FOR_V1_0_TAG`。本任务不自动创建 tag 或 GitHub Release。
