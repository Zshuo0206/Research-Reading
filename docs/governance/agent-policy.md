# Agent Policy

## 角色

| 角色 | 可以做 | 不可以做 |
|---|---|---|
| 人类项目负责人 | 批准范围、RFC/ADR、破坏性操作、发布 | 无 |
| 主控 Agent | 创建直接子 Agent、拆任务、分配 ownership、集成、复跑测试、更新状态 | 创建下级层级之外的 Agent；绕过批准扩大范围 |
| 子 Agent | 执行一个任务、修改声明路径、运行测试、提交交接 | 创建子 Agent、合并代码、修改公共契约、修改迁移、删除未知文件 |
| QA Agent | 审查 diff、运行质量检查、报告问题 | 自行改变被测实现 |

## 强制规则

- 只有主控 Agent 可以创建子 Agent；子 Agent 不得创建下级 Agent。
- 一个子 Agent 只执行一个可验收任务。
- 同一 Wave 同一文件只能有一个写入 owner。
- 子 Agent 不得自行合并、rebase 共享分支或强推远端。
- 失败时交接状态必须是 `PARTIAL` 或 `BLOCKED`，不能用“完成”掩盖阻塞。
- 主控 Agent 必须重新运行关键测试；不能把子 Agent 的输出当作验收证据。
- Agent 发现接口或范围问题时提交 RFC，不直接修改共享契约。
