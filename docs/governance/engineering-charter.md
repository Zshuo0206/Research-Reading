# Engineering Charter

## 目标

本项目以“可追溯、可恢复、可验证、边界清晰”为工程治理目标。Wave 0 先验证协作制度，再开发产品能力。

## 不可违反的原则

1. 接口先于实现，任务先于编码。
2. 单一责任人维护公共契约、数据库迁移和依赖版本。
3. Agent 的输出必须能通过仓库文件和测试复核。
4. 不可信文档、模型输出和外部服务返回值都不能直接成为系统事实。
5. 用户确认前的分析只能是草稿，不能进入批准资产。
6. 所有时间以 UTC 存储和传输，展示层再转换时区。
7. 破坏性操作默认禁止，必须经过人工批准并记录回滚方案。

## 决策层级

人类项目负责人 > 主控 Agent 的集成裁决 > 子 Agent 的局部实现决定。任何下级决定不得扩大上级已冻结的范围。

## Wave 门槛

Wave 只有在目标、任务、ownership、质量门槛、状态记录和退出条件都明确后才能开始。Wave 完成后必须更新 `docs/CURRENT_STATE.md` 和 devlog，并等待人工验收，不自动进入下一 Wave。
