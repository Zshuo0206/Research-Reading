# Engineering Charter

## 目标

本项目以“可追溯、可恢复、可验证、边界清晰”为工程治理目标。Wave 0 先验证协作制度，再开发产品能力。

## 不可违反的原则

1. 接口先于实现，任务先于编码。
2. 单一责任人维护公共契约、数据库迁移和依赖版本。
3. Agent 的输出必须能通过仓库文件和测试复核。
4. 不可信文档、模型输出和外部服务返回值都不能直接成为系统事实。
5. 用户确认前的分析只能是草稿，不能进入批准资产。
6. 所有时间以 UTC 存储和传输，展示层再转换时区。
7. 破坏性操作默认禁止，必须经过人工批准并记录回滚方案。

## 决策层级与验收分工

- 在已批准产品范围内，技术实现、测试、Git/worktree、CI、接口一致性和工程证据由责任 Agent、QA Agent 和主控 Agent负责；主控 Agent对技术验收结论和证据完整性负责。
- 人类项目负责人负责产品目标、范围偏移、用户价值、科研可信度、数据安全、成本、长期产品形态及重大架构/RFC/破坏性操作决策，不承担逐项工程复核。
- 任何 Agent 都不得扩大人类已批准的产品范围。超出范围或影响重大事项时，必须提交人类项目负责人决策。

## Wave 门槛

Wave 只有在目标、任务、ownership、质量门槛、状态记录和退出条件都明确后才能开始。Wave 结束时先由责任 Agent、QA Agent 和主控 Agent完成技术验收，主控提交人类验收摘要并更新 `docs/CURRENT_STATE.md` 和 devlog；人类项目负责人依据摘要决定通过、有条件通过或不通过，主控不得自动进入下一 Wave。
