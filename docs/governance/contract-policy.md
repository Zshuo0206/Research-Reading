# Contract Policy

## 契约范围

公共 API、共享 Schema、事件格式、数据库模型、状态机、错误模型、配置键和导出格式都属于契约。

## 变更流程

1. 提交 `docs/templates/rfc.md` 格式的 RFC。
2. 说明兼容性、迁移、消费者、测试和回滚。
3. 主控 Agent 检查 ownership、影响面和技术验收计划；QA Agent检查兼容性和证据。
4. 若属于核心架构、重要 RFC、产品范围、用户价值、科研可信度、数据安全、成本或长期形态变化，提交人类项目负责人审批；普通技术契约变更由契约 owner、QA 和主控按批准流程完成。
5. 契约 owner 更新版本和正式文档。
6. 先合并契约和契约测试，再合并消费者/实现。

未批准的接口变更只能存在于任务分支，不得伪装为兼容实现进入集成分支。
