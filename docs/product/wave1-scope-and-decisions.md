# Wave 1 Product Scope and Decision Topics

状态：`DRAFT_FOR_HUMAN_REVIEW`  
用途：供人类项目负责人审核，不是已批准的开发任务，也不授权开始编码。

## 一、Wave 1 产品目标

验证问题驱动型文献阅读产品的最小产品闭环：用户围绕研究任务建立一个项目，导入一篇文献，选择阅读目的，获得结构化阅读问题，并能查看可追溯的分析草稿。

Wave 1 的重点是验证“带着问题阅读”是否比普通摘要更有用，不追求实现完整科研资产平台。

## 二、建议纳入范围

1. 研究项目的最小创建与编辑。
2. 论文元数据和本地文件的最小导入管理。
3. 阅读目的选择：方法学习、写作支持、课题决策、实验设计。
4. 阅读任务的创建、状态展示和失败提示。
5. 基于 Mock/可替换适配器的结构化问题计划展示。
6. 共享 API、错误模型、任务状态和审计字段的第一版契约。
7. 用户能够确认、修改或拒绝问题计划。
8. 为后续证据提取保留接口边界，但不承诺完成真实证据生成。

## 三、明确不纳入范围

- 真实模型调用和生产级 Prompt 编排。
- 完整 PDF 解析、OCR、表格/公式理解。
- 真实 Evidence 召回和科研结论自动生成。
- Markdown/Excel 正式资产导出。
- 多用户协作、权限体系和私有化部署。
- Zotero、Word、LaTeX 或浏览器插件。
- 自动判断创新性、研究空白或论文质量。
- 任何未经过本轮范围批准的额外产品模块。

## 四、需要人类项目负责人审核的产品决策

1. 是否认可“最小闭环”作为 Wave 1 产品目标。
2. Wave 1 是否允许真实论文文件导入，还是只使用固定评测样本。
3. 首期用户和核心验证指标是什么：完成率、问题可用性、用户修改率，还是其他指标。
4. 四种阅读模式是否全部纳入，或先选择一个主模式。
5. 是否允许在 Wave 1 使用外部模型；若允许，隐私、成本和数据保留边界是什么。
6. 产品是否需要在 Wave 1 支持本地优先或离线文件处理。
7. Wave 1 的长期产品形态优先级：个人 Web 工作台、课题组工具或其他定位。
8. 是否接受“真实证据链暂不实现”的产品限制。

## 五、Wave 1 开始前必须完成的技术决策

这些议题由主控 Agent、QA Agent 和相关责任 Agent提出技术建议；涉及重大产品、成本、安全或长期形态时，再提交人类项目负责人决定。

- 后端/前端技术栈和运行时。
- QA 的实际执行方式：独立 QA Agent，或与实现 Agent 分离的独立审查任务。
- PDF 文件处理方式和安全边界。
- ModelGateway 是否只做 Mock，或接入真实供应商。
- 数据库、对象存储和异步任务是否进入 Wave 1。
- 第一版 API、错误模型、任务状态和审计字段。
- CI 中真实 formatter、lint、typecheck、依赖和安全工具。
- Wave 1 的评测集、人工基准和 Definition of Done。

## 六、批准条件

在以下条件完成前，不得创建真实业务实现任务：

- 人类项目负责人确认产品范围和核心验证目标。
- 重大隐私、成本和长期产品形态事项已有决定或明确延期。
- 主控 Agent 将批准的范围拆成任务，并建立 ownership、契约版本、验收标准和回滚方案。
- QA 执行方式已明确并写入任务和 Wave 计划。
