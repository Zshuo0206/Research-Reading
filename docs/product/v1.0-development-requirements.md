# 科研文献引导式学习平台 V1.0 开发需求基线

状态：`PRODUCT_BASELINE_RECORDED_IMPLEMENTATION_REFINEMENT_PENDING`  
产品来源：[科研文献引导式学习平台 V1.0 产品定义](./versions/科研文献引导式学习平台_V1.0产品定义.md)  
适用范围：Wave 1 后续业务轮次；不等同于编码授权

## 1. 目的与边界

V1.0 要验证“用户先阅读和思考，系统再点评并回到论文证据”的学习闭环。产品保留两个相互独立的入口：

- 快速问答模式：上传论文 → 生成问题 → 用户确认问题 → 生成带证据的参考回答。它是当前已规划的快速路径。
- 引导式学习模式：上传论文 → 输入学习目标 → 选择学习方向 → 查看路线 → 逐题阅读、作答和修正 → 阶段总结。它是 V1.0 的核心路径。

本基线只覆盖个人单用户、`127.0.0.1` 本地 Web 工作台、一篇文本型可提取 PDF 和第一阶段“理解内容”。不因本文件自动解锁 Gate C，不自动创建分支或开始业务编码。

## 2. V1.0 功能需求

### 2.1 论文和学习上下文

| 编号 | 需求 | 验收要点 |
| --- | --- | --- |
| V1-DOC-01 | 用户可以导入一篇文本型科研论文 PDF | 文件版本、页码文本、canonical hash 和提取 profile 可追溯；扫描/OCR/复杂视觉内容拒绝或明确提示 |
| V1-LEARN-01 | 用户可以输入自由文本学习目标 | 目标绑定到一次学习会话，不以全文摘要替代 |
| V1-LEARN-02 | 系统生成 2–3 个候选学习方向 | 每个方向有可读标题和说明；用户只能选择，V1.0 不手动编辑方向 |
| V1-LEARN-03 | 系统展示固定三阶段路线 | “理解内容”可进入；“分析评价”和“迁移应用”显示“即将开放”，不可误进入真实流程 |

### 2.2 逐题学习

| 编号 | 需求 | 验收要点 |
| --- | --- | --- |
| V1-QUESTION-01 | 开始第一阶段时一次生成完整问题集合 | 生成 3–7 道开放式问题，按基础理解到逐步深入排列 |
| V1-QUESTION-02 | 逐题展示问题 | 用户不必预览或确认完整问题清单；当前题完成后由用户主动点击进入下一题 |
| V1-QUESTION-03 | 提交回答或“不会/跳过” | 只接受自由文本；不接受语音、图片、附件或用户自带引用作为必填条件 |
| V1-ANSWER-01 | 提交后返回简短点评 | 只说明是否抓住核心内容及主要遗漏，不输出数值评分、掌握等级或能力曲线 |
| V1-ANSWER-02 | 返回论文内参考答案和证据 | 参考答案只使用当前论文；包含页码、原文片段、稳定证据位置和断言类型 |
| V1-ANSWER-03 | 支持学习修正 | 参考答案持续可见；用户可修改当前最新版回答；不实现完整回答版本历史，也不把修正当作再次测验 |
| V1-ANSWER-04 | 支持完成阶段 | 全部问题处理后生成简短阶段总结，至少包括本阶段学习内容和用户整体回答情况 |

### 2.3 快速问答兼容

| 编号 | 需求 | 验收要点 |
| --- | --- | --- |
| V1-QUICK-01 | 快速问答模式继续保留 | 该模式可独立完成问题计划确认/修改/拒绝和至少一个带证据回答的确认；不与引导式学习的逐题体验混用 |
| V1-QUICK-02 | 模式边界由服务端强制 | 客户端不能伪造阅读模式、阶段状态、确认状态或证据验证结果 |

## 3. 证据、科研可信度与数据规则

- 每条回答断言必须区分 `PAPER_FACT`、`AUTHOR_CLAIM`、`AGENT_INFERENCE`。
- Evidence 必须由服务端绑定论文文件版本、页码、canonical page text hash、extraction profile 和稳定坐标；模型不能直接提交最终字符 offset。
- 无法由当前论文支持的回答进入明确的无证据/失败状态，不使用外部检索或模型通用知识补充。
- 未经用户确认的快速问答回答不进入正式研究资产；V1.0 不建立正式资产库。
- 外部模型仍采用 BYOK + Mock；发送前要有本地操作员告知/同意，API Key 只存在环境变量或当前会话内存，日志不保存原文、密钥或完整 prompt/response。

## 4. 页面与交互验收

学习页面必须提供左右分栏：

- 左侧显示原始 PDF 页面，支持翻页和跳转到证据页码。
- 右侧显示学习方向、阶段、当前问题、回答输入、提交/不会、点评、参考答案、证据、修改和下一题。
- `loading`、`empty`、`error`、`retry`、`success`、重复提交 `pending` 状态可见且不会产生重复副作用。
- 下一题由用户主动触发；系统可以提示，但不能自动跳题。

## 5. 公共契约与状态机变更

本基线涉及公共契约和状态机调整，必须先完成 [RFC-W1-002](../rfcs/RFC-W1-002-guided-learning-v1-product-alignment.md) 的独立技术审查，再更新 `wave1.v1` 或新建兼容版本。候选字段和状态包括：

- `learning_goal`、`candidate_directions`、`selected_direction`；
- `learning_route`、`current_stage`、阶段开放状态；
- `question_order`、当前题指针、`SKIPPED`/已作答结果；
- `feedback`、参考答案、Evidence 和阶段总结；
- 快速问答与引导式学习的 `mode` 分支以及各自允许的状态转换。

在契约冻结前，任务只能完成设计、fixture 和测试计划，不能自行发明字段或修改共享 Schema。

## 6. 任务拆分与执行顺序

1. 复用已集成的 SQLite/Job、文本 PDF、canonical page text、Evidence 坐标校验和 Mock Gateway。
2. 完成 RFC-W1-002 审查；随后由契约 owner 更新兼容 Schema、类型、fixture 和契约测试。
3. 工作流后端实现快速问答兼容路径，以及学习目标、候选方向、三阶段路线、逐题回答/跳过、点评、参考答案、修正和阶段总结。
4. Web 实现左右分栏的引导式学习页面，并保留快速问答入口。
5. QA 使用固定许可论文独立验证空库、成功、无证据、失败、重试、重复提交和完整 V1.0 浏览器闭环。
6. 主控复跑关键门槛并形成外部交接摘要；产品范围和重大风险仍按 Gate 规则由人类项目负责人决定。

对应任务：`T-W1-011`（契约细化）、`T-W1-005`（工作流 API）、`T-W1-006`（Web 工作台）、`T-W1-007`（评测与 QA）、`T-W1-008`（集成与发布 Gate）。这些任务保持 `DRAFT/LOCKED`，不代表已授权编码。

## 7. 明确不做

V1.0 不实现 OCR、扫描 PDF、复杂表格/公式/图片理解、多篇联合学习、第二/第三阶段、动态难度、追问、外部知识检索、选择题、语音/图片/附件回答、评分/掌握等级、完整学习历史、回答版本历史、复杂学习报告、移动端、协作、导出、资产库、远程部署和 UI 精修。

## 8. V1.0 最低验收场景

在本地网页中从空库开始：导入许可的文本型 PDF，输入学习目标，选择候选方向，查看三阶段路线，完成第一阶段 3–7 道理解题；每题可阅读、作答或跳过，获得简短点评、带页码和原文片段的参考答案，修改当前回答并主动进入下一题；最后看到阶段总结。快速问答模式另行完成其问题和回答确认闭环。
