# CI Gates

## 必过门槛

- 仓库结构检查
- JSON/YAML/Markdown 模板结构检查
- 格式化检查
- lint
- 类型/静态检查
- 基础单元/烟雾测试
- 共享契约 Schema 检查
- 数据库迁移检查（领域模型启用后）
- 构建检查
- secrets 和依赖扫描

## 门槛原则

任何必过门槛失败都不能合并，除非有明确、可追溯的例外批准。责任 Agent 执行局部检查，QA Agent独立复核适用门槛，主控 Agent重新运行关键门槛并在技术验收摘要中记录结果。当前 Wave 0 的占位脚本必须真实执行；尚未实现的产品质量项只能在报告中标明 `NOT_APPLICABLE_BY_SCOPE`，不能声称已覆盖。人类项目负责人不需要逐项运行 CI 或检查具体日志，只接收主控摘要和证据链接。
