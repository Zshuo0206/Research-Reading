# Testing Strategy

## 层级

1. **单元测试**：私有函数、Schema、状态转换和错误分类，快速且确定。
2. **集成测试**：模块与数据库/队列/适配器边界；使用隔离测试资源。
3. **契约测试**：OpenAPI/共享 Schema、错误模型、版本兼容性。
4. **端到端测试**：从用户入口到可观察结果；Wave 0 只覆盖健康端点和占位启动。
5. **AI 评测**：固定输入、参考答案、证据位置、分类边界、Schema 有效率和拒答行为；禁止只用模型自己生成的答案作为唯一基准。

## 额外检查

每次 CI 执行格式化检查、lint、类型/静态检查、基础测试、Schema 校验、迁移检查、构建检查、安全扫描和依赖检查。工具不可用必须显式失败或记录为未执行，不得静默跳过。

Agent 声称完成不能视为验收；主控 Agent/QA 必须重新运行任务要求的关键命令。
