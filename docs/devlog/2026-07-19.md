## 实现事实

- PDF/Evidence Web：iframe key 包含 source/page/reload token；新页和同页重复点击都重建 iframe；URL 始终只有一个 `#page`；显示当前目标、加载/失败状态和新标签页降级入口。
- Web E2E：GET 不再改变 Worker 状态；后台推进由独立 timer 模拟；transition 保存真实 from/to；覆盖第二题 CONFIRM 成功、ADVANCE 临时失败后从 QUESTION_COMPLETED 手动恢复。
- Evidence：Mock Guided Feedback 与真实模式共用 exact-match resolver；PAPER_FACT 只陈述可逐字核对 quote；短或不可唯一定位文本返回 INSUFFICIENT_EVIDENCE；Evidence ID 按题序号稳定隔离。UI 明确 VERIFIED 只代表引用位置和字符串核验。
- API：全局、路由、解析、PDF、Workflow、Guided Learning 和 BYOK 错误统一为 `api.v1` envelope，不发送 Error 对象或内部路径；PDF 路由区分无效 ID 400、不存在 404、存储不可用 500，并只服务完整 PDF。
- BYOK：浏览器 session-key 默认关闭，显式 `ENABLE_BROWSER_SESSION_KEY=1` 才启用；V1 默认仍为 API/Worker 各自读取 `WORKFLOW_BYOK_API_KEY`。
- Worker：ready 在数据库/migration/handler 初始化后输出；runOnce 异常隔离并做 250ms–5s 有界退避；支持 AbortSignal、当前 Job 完成后退出、单次 DB close、结构化脱敏 retry 日志；明确单 Worker 推荐和多 Worker 未做压力认证。
- 真实 smoke：临时 SQLite/content root、真实 synthetic PDF、随机 loopback HTTP、实际 API 和 Worker，完成三题、Evidence、完整 PDF 响应和阶段总结，finally 清理。
- Windows ops：start/check/stop 只管理状态文件中 PID+启动时间匹配的进程；启动失败清理自身进程；不输出 secret；stop 保留数据库/content。首次演练发现 Web cwd 错误并正确清理，修复后 start/check/stop 实际通过。

## 2026-07-22 — 第二轮定向修复

### 实现事实

- Managed stop：start 为每次运行建立 `tmp/v1-local/runs/<run-id>/worker.stop`；Worker 仅轮询文件是否存在并触发既有 `requestStop("CONTROL_FILE")`，不读取内容。当前 `runOnce()` 完成后 loop 才退出，不再领取下一 Job；stop 超时不强杀 Worker、不停止 API/Web、不删除状态。成功路径顺序为 Worker → API → Web。未修改 Storage，也未引入 lease/orphan recovery；非 managed 强杀、系统崩溃和断电后的 RUNNING 恢复仍不支持。
- 固定端口：删除 start 的 ApiPort/WebPort 参数，固定 API 4310、Web 4173。启动任何进程前按 TCP listener 检查端口；冲突时返回非零，不终止占用者且不创建有效状态。
- Worker ready：start/check 对本次 run 独立 stdout/stderr 逐行 `ConvertFrom-Json`，精确核验 ready event、`accepts_jobs=true`、database/migrations/handlers，并识别 start/error/stopped/retry 事件。check 明确输出 healthy/degraded/unhealthy。
- API：新增共享 allowlist classifier；普通对象上的字符串 code 可安全映射到 400/404/409，未知 code 统一为无内部信息的 500 `INTERNAL_ERROR`。Workflow 的既有安全分类一并复用，所有响应保持 `api.v1` 与空 details。
- Evidence：模型原始 `reference_answer` 仅参与输出结构校验，不再直接持久化或展示。最终正文只由 resolved claims 构建：supported 使用“有原文支持的参考内容”，mixed 追加“当前证据不足”，all-IE 使用固定保守提示。UI 将 resolved claims 分为“有原文支持”和“当前证据不足”，模型 summary 标记为非原文验证；Evidence gate 与 exact-match 未变。
- `test:v1-smoke` 的准确边界为真实 HTTP + Worker runtime handlers + SQLite + PDF；它直接调用 runtime handler，不是正式 Worker 主进程 smoke。
